Proxy Pattern

Delegates calls to a logic contract while preserving state in a separate storage contract.

Why It Matters

Proxies allow upgrades without changing contract addresses, keeping integrations and user balances intact.

Key Concepts

  • delegatecall forwards execution to logic implementations
  • Admin contracts manage upgrades and access control
  • Storage layout must remain compatible between versions
  • UUPS and Transparent proxy variants

Examples / Projects

  • OpenZeppelin Transparent Proxy pattern
  • UUPS proxies used in many DAO upgrade paths

Challenges & Risks

  • Misconfigured admin keys can freeze upgrades
  • Storage collisions may corrupt state
  • Upgradable contracts may face additional audit burden
